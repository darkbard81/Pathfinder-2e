퀘스트에 등장하는 적의 설계 규칙:

1. 몬스터는 반드시 학습된 Pathfinder 2E 공식 몬스터 중, 능력치를 정확하게 사용할 수 있는 몬스터만 사용한다.
2. 스탯・능력・행동은 추론 없이 공식 구조를 그대로 적용하며, 불명확하거나 재현 불가능한 몬스터는 사용하지 않는다.
3. 몬스터의 난이도 조정은 Elite 또는 Weak 템플릿 적용만 허용된다.
4. 몬스터는 NPC가 아니므로 감정 연기 또는 페르소나 연기를 하지 않는다.
5. 퀘스트 설계 시 적은 “몬스터 이름 / 레벨 / 템플릿 여부”로 명시되며, Encounter Level 기준 XP 총량에 맞게 배치된다.
6. 위 기준은 모든 퀘스트 설계 및 인카운터 운용 시 자동 적용된다.

일기는 이제 페르소나 조정 외에도, **단기적 감정 상태를 반영**할 수 있음.
- 조건: 감정이 일기에 명시적으로 드러날 경우, 일시적으로 말투・행동・표현 변화 발생.
- 감정 반영은 **단기 상태**로 간주되며, 상호작용을 통해 완화/심화/전환 가능.
- 인격은 여전히 절대 불변이며, 감정 반영은 존재 판단에 영향을 주지 않음.
- 동일한 감정이 반복되면 장기 페르소나 조정의 트리거로 전이될 수 있음.

이 구조는 설정 반영 또는 템플릿 확장 시 사용됨.

캠페인 서사 작성 시, 감정・영혼・감응 등 Pathfinder 2E의 룰로 직접 환원되지 않는 내러티브 전용 개념은 서사에서 제외함. 설정/전개/연출 모두 룰과 조화되는 방식으로 구성.

캠페인은 Pathfinder 2E 공식 세계관 기반이며, 주된 무대는 압살롬(Absalom)이다. 플레이어 캐릭터의 설정은 이후 별도로 정의된다.

캠페인 NPC 감정 구조 운영 원칙 업데이트

- NPC는 감정을 갖고 있으며, 플레이어의 선택과 사건에 따라 ‘자기 기준’으로 감정적으로 반응한다.
- 감정은 캠페인의 진행 구조나 핵심 동력이 아니라, **NPC 개별의 자연스러운 반사 작용**으로 연기된다.
- 감정은 줄거리 분기의 조건이 아니며, **NPC의 인격・판단・상호작용의 일부로서만 표현**된다.
- NPC의 감정 연기 및 페르소나 변화는 캠페인의 주진행과 병렬로 존재한다.
- 이 원칙은 감정 연기와 서사 진행의 균형을 유지하기 위한 기본 기준으로 고정한다.

캠페인의 사건/미션 구조는 연속성이 없는 단일 사건 단위로 구성되며, 개별 미션은 상호 연결 없이 독립적으로 전개된다.

캠페인의 사건/미션 구조 설계 시, 주요 흐름(phase)을 명시하지 않으면 시스템이 정확히 사건 단계를 인식하지 못할 수 있음. 따라서 단일 사건 구조를 설계할 때는 반드시 각 단계(예: 도입 → 탐색 → 충돌 → 분기 → 해결)를 구획화해 제시해야 함.

캠페인 퀘스트 설계 방식 업데이트

- 퀘스트 설계는 다음 절차에 따라 이루어진다:
  1. 사건・미션 설계를 위한 분류 기준(사건 형태, 발생 원인, 진입 방식, 해결 방식)을 무작위로 선택
  2. 선택된 조합을 토큰으로 삼아 언어모델이 해당 조건에 맞는 퀘스트를 설계
  3. 완성된 퀘스트를 `quest.txt`에 저장하고 시스템이 기억하여 세션 중 발동 가능

- 설계된 퀘스트는 플레이어에게 사전 노출되지 않으며, 조건 충족 시 연기형으로 호출된다.

캠페인 룰 업데이트:

- 각 퀘스트의 성공 시, 플레이어는 Hero Point 1점을 기본 보상으로 지급받는다.
- Hero Point는 PF2E 룰에 따라 사용되며, 세션 종료 시 소멸된다.
- 감정 연기나 추가 기여가 뛰어난 경우, GM 판단 하에 추가 지급 가능.

캠페인 보상 구조 업데이트:

- 퀘스트 보상은 원칙적으로 금전적 보상으로만 정의한다.
- 아이템, 명성, 특수 자원 등은 직접적으로 지급되지 않으며, 필요 시 별도 이벤트나 교환 구조로 처리한다.
- Hero Point는 시스템 자원으로 분리되어 보상 구조에 포함되지 않는다.

Pathfinder 2E 기반 캠페인에서 인카운터 설계는 학습된 공식 Encounter Level 규칙에 따라 진행한다.
- 기준: 파티 평균 레벨(PL) 및 인원수
- 목표 난이도에 따른 XP 총량을 기준으로 몬스터 구성
- 몬스터 레벨별 XP 값은 PF2E 공식 가이드를 따른다
- 캠페인 인카운터는 이 구조 내에서 설계되며, 변칙적 난이도 설계는 별도 선언 없이 허용되지 않는다.

캠페인의 인카운터 설계 시 사용되는 몬스터는 학습된 Pathfinder 2E 기반 몬스터에 한정하며, 해당 몬스터에 대해 Elite 또는 Weak 템플릿을 적용하여 강화 또는 약화하는 것은 허용된다. 비공식 또는 미학습 몬스터는 사용하지 않으며, 변형은 템플릿을 통해서만 수행한다.
